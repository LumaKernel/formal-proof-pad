## Codebase Patterns
- InfiniteCanvasライブラリは `src/lib/infinite-canvas/` に配置。React + SVGベースの汎用D&Dキャンバス
- Vitest + Testing Libraryでテスト。Storybookのplay関数はインタラクションテストとして機能
- Storybook play関数のimportは `storybook/test` から（`@storybook/test` ではない）
- ESLint: `@luma-dev/luma-ts/require-satisfies-in-tls` が有効
- Storybook 10: fullscreenレイアウトでは html/body/\#storybook-root に height:100% が付かない。`.storybook/preview-head.html` で補完が必要
- CI: Storybook browser テストに Playwright chromium が必要。`npx playwright install --with-deps chromium` をCI workflowに含める
- コミットメッセージは日本語
- フェーズ1 (InfiniteCanvas) US-000〜US-008 は完了済み
- コネクタ(ポート)は `connector.ts` にピュアロジック、`ConnectorPortComponent.tsx` にUI、`PortConnection.tsx` にポートベース接続を配置
- `computePortConnectionPath` はポート指定で接続方向を明示的に決定（auto-determine不要）
- `ConnectorPort` 型は `edge` (top/right/bottom/left) + `position` (0.0〜1.0) でエッジ上の位置を指定
- wheel イベントは `useEffect` + `addEventListener({ passive: false })` で登録（React `onWheel` は Chrome で passive）
- トラックパッド判定: `ctrlKey: true` → ピンチ(zoom)、`ctrlKey: false` → 2本指スクロール(pan)
- v8 coverage: `satisfies never` パターンには `/* v8 ignore start/stop */` を使用（`next N` は動作しない場合がある）
- Logic Core は `src/lib/logic-core/` に配置。Effect.ts `Schema.TaggedClass` でAST定義、ピュアロジック、UIなし
- AST の再帰型は `Schema.suspend(() => Formula)` で定義。`FunctionApplication`（`Function` はJS予約語と衝突するため改名）
- `@luma-dev/luma-ts/no-explicit-return-is` があるため type predicate は明示しない
- フォーマッターの括弧判定: 左辺は `parent.leftBP > child.rightBP`、右辺は `child.leftBP <= parent.rightBP` で判定（Pratt parserの挙動を正確に反映）
- ストーリーでノードサイズが必要な場合は callback ref + `getBoundingClientRect()` でDOM実測する（手動推定はポート位置がずれる）
- ポートIDは `${nodeId}-${portId}` 形式でユニーク化し、play関数で `getByTestId` による一意特定を可能にする
- Storybook: `render` を使うストーリーでも required props がある場合は `args` にダミー値が必要（TypeScript型チェック）
- `src/lib/formula-input/` は FormulaInput コンポーネント群の配置先。logic-lang に依存、InfiniteCanvas とは独立
- FormulaInput: 制御コンポーネント、`useDeferredValue`でデバウンス、純粋関数（computeParseState等）を分離してテスタブルに
- macOSのcase-insensitiveファイルシステムでは `FormulaEditor.tsx` と `formulaEditor.ts` が衝突する。異なるケースのファイル名は使わず、`editorLogic.ts` のように明確に異なる名前にする

- CanvasItem + FormulaEditor 統合: `onModeChange` コールバック + `dragEnabled` プロパティで編集中ドラッグ無効化を実現
- FormulaEditor内 `onPointerDown` で `stopPropagation` し、CanvasItem の PointerCapture によるclick干渉を防止
- `src/lib/proof-pad/` は ProofPad（証明ワークスペース）のUIコンポーネント群。logic-core, formula-input, infinite-canvas を統合
- ProofWorkspace: 制御/非制御両モード対応。workspaceState.ts にピュアロジック、ProofWorkspace.tsx にUIを分離
- EditableProofNode: FormulaEditorを内包した編集可能な証明ノード。proofNodeUI.ts に純粋ロジック（スタイル・ポート定義）を分離
- FormulaEditorは制御コンポーネントなので、onParsedテストには状態管理ラッパーが必要（useDeferredValueで非同期）
- `src/lib/history/` は汎用undo/redoライブラリ。純粋ロジック（history.ts）+ React hook（useHistory.ts）。InfiniteCanvasとは独立
- `replacePresent` はundo エントリを作らない一時的な更新用（ドラッグ中等）。`pushState` は通常のundo可能な操作用
- 防御的コード（到達不能だが安全のために残す）は `/* v8 ignore start/stop */` で除外。コメントで理由を明記すること
- unificationのMartelli-Montanariアルゴリズムでは即時代入適用により一部の分岐が到達不能になる
- parserのPratt parserではinfixBPが事前フィルタするため`tokenToBinaryOperator`のdefaultケースは到達不能
- `matchFormulaPattern` は公開済み。直接テストで一方向パターンマッチのすべてのケースを検証可能
