## Codebase Patterns
- InfiniteCanvasライブラリは `src/lib/infinite-canvas/` に配置。React + SVGベースの汎用D&Dキャンバス
- Vitest + Testing Libraryでテスト。Storybookのplay関数はインタラクションテストとして機能
- Storybook play関数のimportは `storybook/test` から（`@storybook/test` ではない）
- ESLint: `@luma-dev/luma-ts/require-satisfies-in-tls` が有効
- Storybook 10: fullscreenレイアウトでは html/body/\#storybook-root に height:100% が付かない。`.storybook/preview-head.html` で補完が必要
- CI: Storybook browser テストに Playwright chromium が必要。`npx playwright install --with-deps chromium` をCI workflowに含める
- コミットメッセージは日本語
- フェーズ1 (InfiniteCanvas) US-000〜US-008 は完了済み
- コネクタ(ポート)は `connector.ts` にピュアロジック、`ConnectorPortComponent.tsx` にUI、`PortConnection.tsx` にポートベース接続を配置
- `computePortConnectionPath` はポート指定で接続方向を明示的に決定（auto-determine不要）
- `ConnectorPort` 型は `edge` (top/right/bottom/left) + `position` (0.0〜1.0) でエッジ上の位置を指定
- wheel イベントは `useEffect` + `addEventListener({ passive: false })` で登録（React `onWheel` は Chrome で passive）
- トラックパッド判定: `ctrlKey: true` → ピンチ(zoom)、`ctrlKey: false` → 2本指スクロール(pan)
- v8 coverage: `satisfies never` パターンには `/* v8 ignore start/stop */` を使用（`next N` は動作しない場合がある）
- Logic Core は `src/lib/logic-core/` に配置。Effect.ts `Schema.TaggedClass` でAST定義、ピュアロジック、UIなし
- AST の再帰型は `Schema.suspend(() => Formula)` で定義。`FunctionApplication`（`Function` はJS予約語と衝突するため改名）
- `@luma-dev/luma-ts/no-explicit-return-is` があるため type predicate は明示しない
- フォーマッターの括弧判定: 左辺は `parent.leftBP > child.rightBP`、右辺は `child.leftBP <= parent.rightBP` で判定（Pratt parserの挙動を正確に反映）
- ストーリーでノードサイズが必要な場合は callback ref + `getBoundingClientRect()` でDOM実測する（手動推定はポート位置がずれる）
- ポートIDは `${nodeId}-${portId}` 形式でユニーク化し、play関数で `getByTestId` による一意特定を可能にする
- Storybook: `render` を使うストーリーでも required props がある場合は `args` にダミー値が必要（TypeScript型チェック）
- `src/lib/formula-input/` は FormulaInput コンポーネント群の配置先。logic-lang に依存、InfiniteCanvas とは独立
- FormulaInput: 制御コンポーネント、`useDeferredValue`でデバウンス、純粋関数（computeParseState等）を分離してテスタブルに
- macOSのcase-insensitiveファイルシステムでは `FormulaEditor.tsx` と `formulaEditor.ts` が衝突する。異なるケースのファイル名は使わず、`editorLogic.ts` のように明確に異なる名前にする

- CanvasItem + FormulaEditor 統合: `onModeChange` コールバック + `dragEnabled` プロパティで編集中ドラッグ無効化を実現
- FormulaEditor内 `onPointerDown` で `stopPropagation` し、CanvasItem の PointerCapture によるclick干渉を防止
- `src/lib/proof-pad/` は ProofPad（証明ワークスペース）のUIコンポーネント群。logic-core, formula-input, infinite-canvas を統合
- ProofWorkspace: 制御/非制御両モード対応。workspaceState.ts にピュアロジック、ProofWorkspace.tsx にUIを分離
- nodeRoleLogic.ts: ノード分類の純粋ロジック。NodeRole("axiom"|"goal") と NodeClassification("root-axiom"|"root-goal"|"root-unmarked"|"derived") で分類
- EditableProofNode: FormulaEditorを内包した編集可能な証明ノード。proofNodeUI.ts に純粋ロジック（スタイル・ポート定義）を分離
- FormulaEditorは制御コンポーネントなので、onParsedテストには状態管理ラッパーが必要（useDeferredValueで非同期）
- `src/lib/history/` は汎用undo/redoライブラリ。純粋ロジック（history.ts）+ React hook（useHistory.ts）。InfiniteCanvasとは独立
- `replacePresent` はundo エントリを作らない一時的な更新用（ドラッグ中等）。`pushState` は通常のundo可能な操作用
- 防御的コード（到達不能だが安全のために残す）は `/* v8 ignore start/stop */` で除外。コメントで理由を明記すること
- MP適用ロジックは3層分離: 純粋バリデーション(`mpApplicationLogic.ts`) → 状態管理(`workspaceState.ts`) → UI(`ProofWorkspace.tsx`)
- Gen適用ロジックも同様の3層: `genApplicationLogic.ts` → `workspaceState.ts` → `ProofWorkspace.tsx`。新しい推論規則追加時はこのパターンに従う
- `formatFormula` は右結合→で最小括弧化。テスト期待値は `χ → φ → ψ → φ`（括弧なし）
- unificationのMartelli-Montanariアルゴリズムでは即時代入適用により一部の分岐が到達不能になる
- parserのPratt parserではinfixBPが事前フィルタするため`tokenToBinaryOperator`のdefaultケースは到達不能
- `matchFormulaPattern` は公開済み。直接テストで一方向パターンマッチのすべてのケースを検証可能
- `treeLayoutLogic.ts`: 証明ツリーの自動レイアウト純粋ロジック。カスタム実装（外部依存なし）。DAG→フォレスト→位置計算
- `applyTreeLayout` in `workspaceState.ts`: ワークスペースノードにレイアウトを適用する統合関数

- 公理パレット: `axiomPaletteLogic.ts`（純粋ロジック）+ `AxiomPalette.tsx`（UI）のペア。case-insensitive FS対策で`...Logic.ts`と`...ette.tsx`で名前区別
- ゴールチェック: `goalCheckLogic.ts`（純粋ロジック）でゴール式とワークスペースノードの一致判定。`WorkspaceState`に`goalFormulaText`フィールド追加
- テーマ管理: `src/lib/theme/` に3層アーキテクチャ（themeLogic.ts純粋ロジック + useTheme.ts hook + ThemeProvider.tsx Context）。matchMediaモックパターンに注意
- Storybook import: `@storybook/nextjs-vite` を使用（`@storybook/react` ではなく）
- テーマCSS変数は `globals.css` に `[data-theme="light"]` / `[data-theme="dark"]` セレクタで一元定義。CSS Modulesからはグローバル変数を参照する
- テーマ遷移: `--theme-transition-duration` CSS変数で一元管理。`data-theme-loaded` 属性でFOUC防止、`prefers-reduced-motion` で無効化
- FOUC防止: `layout.tsx` の `<head>` にインラインスクリプトでlocalStorageからテーマ読み取り → `data-theme` 属性設定。React hydration前に実行
- InfiniteCanvasのデフォルト色はCSS変数 `var(--color-canvas-bg, #ffffff)` / `var(--color-canvas-dot, #c0c0c0)` で指定。propsで上書き可能
- InfiniteCanvasのグリッドライン: SVGパターン重ね描画。`computeGridLinePatternParams` で純粋計算、`majorGridEvery=0` で無効化
- Storybook iframe内でCSS変数を使うには `.storybook/preview.ts` で `globals.css` をインポートする必要がある

- Storybookテーマ切り替え: `globalTypes` + デコレーターで `data-theme` 属性を制御。ThemeProviderは使わず、CSS変数の `[data-theme]` セレクタに直接依存
- Side by Sideモード: ストーリーを2回レンダリングするため、play関数のクエリが複数マッチする。テストは単一テーマ（デフォルト: light）で実行
- FormulaEditorのホバースタイルは`currentColor`ベース。不透明な背景色はEditableProofNode内（白文字+色付き背景）でコントラスト問題を起こすため禁止
- パーサーは `phi`, `psi`, `chi` 等のギリシャ文字名を命題変数として認識する。`p`, `q` 等の単一文字は項変数として扱われ、`=` が期待される
- コピペロジックも3層分離: `copyPasteLogic.ts`(純粋データ変換) → `workspaceState.ts`(状態統合) → `ProofWorkspace.tsx`(UI・クリップボードAPI)
- UIオーバーレイ（バナー等）のclickにはstopPropagationが必要（コンテナのclick解除を防ぐ）
- マルチセレクションも3層分離: `multiSelection.ts`(純粋ロジック) → `useMarquee.ts`(React hook) → caller統合
- InfiniteCanvasの`panEnabled`でパン/マーキーモード切り替え。空領域イベントは`onEmptyArea*`で転送
- キーボードショートカットも3層: `keyboardShortcuts.ts`(純粋ロジック) → `useKeyboardShortcuts.ts`(hook) → caller統合
- テストで`as unknown as`を避けるため、hookのイベント型は`KeyboardEventLike`（narrower type）で定義
- `screenToWorld`: `(screenX - offsetX) / scale` でスクリーン→ワールド変換（worldToScreenの逆）
- 補完候補の重複排除: `computeCompletions` で同じ `insertText` の候補を `Set` で重複排除する
- Prettier: 空のmdファイルは0バイトにする（改行のみの1バイトは不正フォーマット扱い）
- `*.local.*` ファイルは `.gitignore` 済みだが、ESLint configにも `**/*.local.*` で除外が必要
- 新ファイル作成後は必ず `npx prettier --write` を実行する（特にmdファイル）
- `src/lib/notebook/` はノートブック管理の純粋ロジック。notebookState.ts にNotebook/NotebookCollection型と操作関数を配置
- ノートブック管理も3層分離パターン: notebookState.ts(純粋ロジック) → hook → UIコンポーネント
- LogicSystemのReadonlySet<PropositionalAxiomId>はJSON化不可。シリアライズ時にArray変換、デシリアライズ時にSet復元が必要
- `@luma-dev/luma-ts/no-date` ルールのため、hookでは `getNow` DI パターンで Date.now() 依存を注入。テスト時はインクリメントカウンタで決定的タイムスタンプ
- 日付フォーマットは `Intl.DateTimeFormat` を使用（`new Date()` はno-dateルール違反）
- localStorage永続化hookパターン: useState初期化でloadし、useEffectでsave（useTheme.tsと同じ）
- `src/lib/quest/` はクエストモード管理。questDefinition.ts(型・カテゴリ) + builtinQuests.ts(問題データ) + questProgress.ts(進捗) + questCatalog.ts(カタログ)
- クエスト進捗のMapもJSON化不可。serialize/deserializeパターンはnotebookSerializationと同様
- ノード検索も3層分離: nodeSearch.ts(純粋) → useNodeSearch.ts(hook) → NodeSearchComponent.tsx(UI)
- キーボードショートカット追加時は `KeyboardAction` union + `classifyKeyDown` + `handleAction` switch の3箇所を変更
