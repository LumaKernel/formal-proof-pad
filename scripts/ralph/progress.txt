## Codebase Patterns
- ESLintはFlat Config (`eslint.config.mjs`) を使用、eslint v9+
- `@luma-dev/luma-ts` プラグインで厳格なTSルール: `no-as-unknown-as`, `no-date`, `no-explicit-return-is`, `prefer-immutable`, `require-satisfies-in-tls`
- テンプレートリテラル内の式には `satisfies` が必要（例: `${value satisfies string}`）
- 明示的な `is` 型述語は禁止 → `find()` パターンで型ナローイングする
- プロパティには `readonly` を付ける
- Prettierフォーマット: `npm run lint-fix` で適用
- テスト: Vitest + jsdom (unit), Storybook + Playwright (browser)
- カバレッジ: v8プロバイダ、`npx vitest run --coverage --project unit`
- infinite-canvasライブラリ: 純粋ロジック(grid.ts, pan.ts, zoom.ts, coordinate.ts, drag.ts) + Reactフック(usePan.ts, useZoom.ts, useDragItem.ts) + コンポーネント(InfiniteCanvas.tsx, CanvasItem.tsx)の分離パターン
- アイテムドラッグ: `stopPropagation()`でキャンバスパンと区別、`useDragItem`フックでスクリーンデルタ→ワールドデルタ変換（scale除算）
- CanvasItem: ワールド座標→スクリーン座標変換は `worldToScreen()` 純粋関数で、CSSのabsolute + transform scaleで配置
- Storybookのplay関数: `storybook/test` からimport（`@storybook/test` ではない）
- ズームロジック: `applyZoom`はカーソル中心ズーム（ワールド座標固定）、`computeScaleFromWheel`は指数的スケーリング
- ピンチズームはPointerEventsで2ポインタを追跡、距離変化からスケール変化を算出
- `react-hooks/refs`ルール: テストハーネスでもrefをコールバックに渡すとエラーになるので注意
- レンダーに必要な状態はuseState、イベントハンドラ内のみの状態はuseRefで管理（react-hooks/refsルール対応）
- SVGパターンでドットグリッド描画（個別dot描画よりパフォーマンス良好）
- jsdomテストでstyle.backgroundColorは`rgb(r, g, b)`形式で返る（hex不可）
- macOS（大文字小文字非区別FS）では `Foo.tsx` と `foo.ts` が衝突する → 純粋ロジックとコンポーネントで同名ファイルを作らない（例: `connectionPath.ts` + `Connection.tsx`）
- 接続線: `computeEdgePoint`（ray-box intersection）でアイテムの矩形エッジから出発点を算出、`computeConnectionPath`でベジェ曲線を生成
- Connection: SVGのpath要素で描画、点線(strokeDasharray) + 半透明(opacity) + 背景白線で要素の後ろでも視認可能
