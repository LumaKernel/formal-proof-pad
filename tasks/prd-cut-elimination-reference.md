# PRD: カット除去定理リファレンスドキュメント

## はじめに

`dev/cut-elimination/` 配下に、カット除去定理（Cut Elimination Theorem）に関する知識をまとめたリファレンスドキュメントを作成します。シーケント計算の基礎から始め、カット規則の定義、カット除去定理の内容、そして実際のカット除去アルゴリズムを疑似コードとコメントによるreasoning付きで記述します。

本プロジェクトのメイン機能（Hilbert系）とは直接統合しませんが、形式証明の重要な背景知識として、将来的な拡張やAIエージェントの論理学理解の補強に活用します。

## 目標

- カット除去定理の理論的背景をAIエージェントが理解できるレベルで記述する
- カット除去アルゴリズムを疑似コード＋コメントで実装可能なレベルまで具体化する
- 各変換ステップの正当性（reasoning）をコメントで明記する
- 参考文献を明示し、原典に辿れるようにする

### US-003: カット除去アルゴリズムの疑似コード

**説明:** AIエージェントとして、カット除去の具体的な変換手順を疑似コードで理解し、将来的に実装可能な知識を得たい。

**受け入れ基準:**

- [ ] `dev/cut-elimination/03-algorithm.md` を作成
- [ ] カット除去アルゴリズムの全体構造を疑似コードで記述:
  ```
  function eliminateCuts(proof: Proof): Proof
  ```
- [ ] 主要な変換ケースを疑似コード＋reasoning（コメント）で記述:
  - **Case 1: 公理とのカット** — カット式が直ちに公理から導かれる場合
    - 疑似コード + なぜこの変換が正しいかのコメント
  - **Case 2: カット式が主式でない場合（Principal reduction ではない）**
    - 左の証明でカット式が主式でない場合
    - 右の証明でカット式が主式でない場合
    - 疑似コード + reasoning
  - **Case 3: カット式が両方で主式の場合（Key case / Principal cut）**
    - 各論理結合子（→, ∧, ∨, ¬, ∀, ∃）ごとの変換
    - 疑似コード + reasoning
  - **Case 4: 構造規則との相互作用**
    - 弱化・縮約とカットの交換
    - 疑似コード + reasoning
- [ ] 各ケースに具体例（変換前 → 変換後の証明図）を付ける
- [ ] **停止性の議論:** なぜアルゴリズムが停止するか（カットの複雑度の減少）をコメントで説明
- [ ] 参考文献へのリンクを含める

## 機能要件

- FR-1: `dev/cut-elimination/` ディレクトリにドキュメント群を配置する
- FR-2: 疑似コードは言語非依存だが TypeScript 風の型注釈を付けて可読性を高める
- FR-3: すべての変換ケースにreasoning（なぜ正しいか）をコメントで付記する
- FR-4: 具体例は証明図をテキストで表現する（Markdown内でインデントや罫線文字を使用）
- FR-5: 各ドキュメントは自己完結型とし、前提知識は他のドキュメントへのリンクで補う
- FR-6: `dev/logic-reference/` のドキュメントを前提知識として参照リンクする

## 非目標

- カット除去定理の完全な形式的証明は含めない（概略とreasoning付き疑似コードに留める）
- カット除去の計算量解析（超指数的爆発の厳密な証明等）は含めない
- 線形論理やその他の非古典論理のカット除去は含めない
- カット除去の実際のTypeScript/Effect.ts実装は含めない（ドキュメントのみ）

## 技術的考慮事項

- **疑似コードのスタイル:** TypeScript風の型注釈付き。パターンマッチはswitch/caseスタイルで
- **証明図の表現:** Markdown内でテキストベースの横線（`────`）とインデントで表現
- **ファイルサイズ:** US-003（アルゴリズム）は長くなる可能性があるため、必要に応じてケース別にファイル分割
- **`dev/logic-reference/` との関係:** シーケント計算はHilbert系と別体系だが、論理記号・量化子の基本知識は共有。相互リンクで参照

## 成功指標

- AIエージェントがドキュメントを読んで、カット除去の各変換ケースを説明できる
- 疑似コードのreasoningコメントにより、各変換がなぜ正しいかが理解できる
- 具体例のステップバイステップを追跡して、アルゴリズムの動作を確認できる
- 将来的にシーケント計算ベースの機能を追加する際のベース知識として機能する
