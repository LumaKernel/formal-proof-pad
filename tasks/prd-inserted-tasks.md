# 差し込みタスク

- [ ] 公理の判別ロジックを修正する。`φ→(φ→φ)` のような代入後の式が「公理K」として検出されてはいけない。
    - well-knownな公理として判別されるのは、公理スキーマそのものの形（例: `φ→(ψ→φ)`）のみ。メタ変数が異なる形で使われている場合のみ公理と判定する。
    - `φ→(φ→φ)` は `φ→(ψ→φ)` に `ψ:=φ` を代入して得られるものであり、公理Kそのものではない。これを公理Kと表示してはならない。
    - ユーザーは `φ→(ψ→φ)` を公理として配置し、代入操作ノードを介して `φ→(φ→φ)` を導くべき。
    - "needs substitution step" という表示も不要。そのような中間状態のノードは存在しない。
    - 参考: http://100.73.163.99:13006/?path=/story/proofpad-proofflowdemo--identity-proof-completed
    - 関連するストーリー・テストの更新
- [ ] 代入操作UIを根本的に見直す。現在のAdd/Delete方式の代入対象指定をやめる。
    - スキーマに含まれるメタ変数は計算で列挙可能なので、代入対象を手動で追加/削除させる必要はない。
    - 代わりに、前提スキーマから自動抽出されたメタ変数の一覧を表示し、それぞれに何を代入するかを入力するだけのUIにする。
    - 参考: http://100.73.163.99:13006/?path=/story/proofpad-proofworkspace--substitution-applied およびsubstitution関連のすべてのストーリー
