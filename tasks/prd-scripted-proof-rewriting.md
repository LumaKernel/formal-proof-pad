# PRD: コード実行による証明書き換え（発展・低優先度）

> **優先度: 低** — 他のPRD（特にlogic-core, logic-lang, ドキュメント整備）が完了した後に着手する発展機能です。

## 追加指示 (適切にタスクリストにまずは反映させて)

- [ ] また、処理が重たくなったり、無限ループになっているときに適切に止めたり気付けたりできるように、多少ゆっくりの実行になってもいいから、ユーザーが介入することができるように、数万ステップごとにちょっと時間おいたり、だんだん実行がゆっくりになるようになったり、とか工夫してほしい

## はじめに

ユーザーがJavaScriptコードを書いて証明図をプログラムで操作できる機能を追加します。JS-Interpreter（Neil Fraser / Google）を使い、完全にサンドボックス化された環境でステップ実行を実現します。Monaco Editorで証明操作APIの補完付きコーディング体験を提供し、テンプレートとしてカット除去定理の実行例を用意します。

これにより、手動のドラッグ&ドロップだけでなく、アルゴリズム的に証明を操作・変換する体験が可能になります。

## 目標

- サンドボックス化されたJavaScript実行環境で証明図の読み取り・書き換え・検証をプログラムから行えるようにする
- ステップ実行により、アルゴリズムの各ステップでの証明図の変化を可視化する
- Monaco Editorでブリッジ済みAPIの補完・型情報を提供する
- テンプレートとしてカット除去定理を実装したサンプルコードを同梱し、実行・ステップ実行で学べるようにする

## ユーザーストーリー

### US-001: JS-Interpreterの統合と基本実行

**説明:** 開発者として、JS-Interpreterをプロジェクトに統合し、サンドボックス内でJavaScriptコードを安全に実行できるようにしたい。

**受け入れ基準:**

- [x] JS-Interpreter（`js-interpreter` npm パッケージ）をインストール・統合
- [x] サンドボックス内でJavaScriptコードを実行する基本ラッパーを作成（`src/lib/script-runner/`）
- [x] `run(code)`: コード全体を実行して結果を返す
- [x] `step()`: 1ステップずつ実行してステップごとの状態を返す
- [x] 実行時間制限（タイムアウト）を設定可能（デフォルト: 5秒）
- [x] ステップ数制限を設定可能（デフォルト: 10,000ステップ）
- [x] ホストのDOM・グローバルに一切アクセスできないことを検証するテスト
- [x] 型チェック/lintが通る

### US-002: 証明操作APIのブリッジ

**説明:** 開発者として、サンドボックス内のコードからlogic-coreの証明操作関数を呼び出せるようにしたい。

**受け入れ基準:**

- [x] JS-Interpreterのブリッジ機構で以下のAPIをサンドボックスに公開:
  - `parseFormula(text)`: テキスト→Formula JSON
  - `formatFormula(formula)` / `formatTerm(term)`: JSON→テキスト
  - `equalFormula(a,b)` / `equalTerm(a,b)`: 構造的等価性
  - `applyMP(antecedent, conditional)`: Modus Ponens
  - `applyGen(formula, variableName, system)`: 汎化規則
  - `unifyFormulas(source, target)` / `unifyTerms(source, target)`: ユニフィケーション
  - `substituteFormula(formula, map)`: メタ変数代入
  - `identifyAxiom(formula, system)`: 公理識別
- [x] ブリッジ関数の引数・戻り値はJSON互換のプレーンオブジェクトに変換
- [x] 不正な操作時は例外をスロー（RuntimeError として伝播）
- [x] ブリッジ関数の一覧を定数として定義（Monaco Editorの補完に使用）— `PROOF_BRIDGE_API_DEFS` + `generateProofBridgeTypeDefs()`
- [x] ユニットテストでブリッジ関数の呼び出しと戻り値を検証（38テスト）
- [x] 型チェック/lintが通る

## 機能要件

- FR-1: JS-Interpreterでサンドボックス化されたJavaScript実行環境を提供する
- FR-2: `step()` メソッドでステップ実行を可能にし、各ステップの状態を取得できる
- FR-3: 証明操作API（取得・書き換え・検証・規則適用・代入・ユニフィケーション）をブリッジする
- FR-4: Monaco Editorでブリッジ済みAPIの補完と型情報を提供する
- FR-5: ステップ実行時に現在の実行行をハイライトし、証明図をリアルタイム更新する
- FR-6: 実行時間・ステップ数の制限で無限ループを防止する
- FR-7: カット除去定理のテンプレートスクリプトを同梱する
- FR-8: ユーザースクリプトをlocalStorageに保存・復元する

## 非目標

- 汎用的なIDEの構築（デバッガのブレークポイント、変数ウォッチ等）は対象外
- Skulpt（Python）のサポートは初期リリースでは対象外（将来的に検討可能）
- スクリプトのサーバーサイド実行・共有は対象外
- TypeScriptの直接実行は対象外（JavaScriptのみ）
- logic-core以外のAPI（DOM操作、ファイルI/O等）のブリッジは対象外

## デザイン考慮事項

- **レイアウト:** 左にMonaco Editor + コントロール、右にキャンバス（証明図）の分割レイアウト
- **ステップ実行:** 現在の実行行が黄色ハイライト、証明図の変更箇所もハイライト
- **テンプレート:** ドロップダウンまたはサイドパネルから選択
- **エラー表示:** エディタ上の波線 + 下部パネルにエラーメッセージ

## 技術的考慮事項

- **JS-Interpreter:** `js-interpreter` npm パッケージ（Neil Fraser / Google）。`step()` メソッドが組み込み。`createPrimitive` / `createNativeFunction` でブリッジ
- **Monaco Editor:** `@monaco-editor/react` を使用。`addExtraLib` でブリッジAPIの型定義（`.d.ts`）を注入し、補完を実現
- **ブリッジのシリアライズ:** JS-Interpreterはプリミティブとオブジェクトの変換が必要。証明図のAST ↔ プレーンオブジェクトの変換レイヤーを設ける
- **パフォーマンス:** ステップ実行時のキャンバス再描画はステップごとではなく、変更があった場合のみ
- **依存関係:** logic-core（US-009〜US-014）が完成していることが前提。カット除去テンプレートには `dev/cut-elimination/` ドキュメントが前提
- **独立性:** `src/lib/script-runner/` はlogic-coreとは独立したラッパー。ブリッジ定義は別モジュール

## 成功指標

- カット除去テンプレートをステップ実行して、各変換ステップが証明図上で可視化される
- Monaco Editorで `getProof()` と入力すると補完候補と型情報が表示される
- ユーザーが自作スクリプトで証明操作を実行し、結果を確認できる
- 無限ループしたコードがタイムアウトで安全に停止する
