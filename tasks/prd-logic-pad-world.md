各種ライブラリ、コンポーネントが完成したら、いよいよ、ノート的に自由に論理を展開していくことができるツールを作成する。
これはのちほど行っていくものになる。また、引き続きコンポーネント、ライブラリベースの発想で開発をしていく。

- [x] ノート管理ができる。複製、削除、変換、詳細を見る、など
- [x] ノートの新規作成では、モードや公理系などが選択できる。
- [-] クエストモード的な形で、基礎を学んだり、問題形式で進めることができる。
  - [x] 大きなゴールはいくつかあるし、学ぶ順番は自由でどこからでも再開したり進めることができる
  - [ ] 大きなゴールのひとつはカット除去定理を遂行すること。実装をサポートしてもらいながら実際にカット除去定理まで学べたらいい
  - [ ] カット除去定理の一個次として、Don't eliminate cutの例を載せたい。 https://www.jstor.org/stable/30226313 (./dont-elim-cut.pdf を参照)
- [x] i18n: 証明に関連する全ての要素において、 MP などの用語を除けば日本語化できるものであってほしい。
  - [x] たとえば、MPの適用が失敗しているだといかいうメッセージはi18nの対象であるべきだろう
- [x] ゴールが設定されている場合、それぞれのノードはゴールとして設定されてるだけでなく、どのような前提のみ使っていいのか、という情報も内部的に持っているべきだろう。
      そうでなければ、使っていい公理だけで達成したか分からない。 (あくまでも内部的だが、ゴールの詳細などを開けば分かるものであるとよいだろうね。side by sideなどで)
