# PRD: ダークモード対応

## はじめに

アプリケーション全体にダークモードを導入します。デフォルトではOS設定に従い、ユーザーが手動でライト/ダーク/システムを切り替えられるトグルUIを設けます。設定はlocalStorageに永続化します。InfiniteCanvasを含む全コンポーネントがテーマに応じた配色になり、切り替え時にスムーズなアニメーション遷移を行います。Storybookでもダーク/ライト両方のプレビューを可能にします。

また、InfiniteCanvasに定期的な基準線（メジャーグリッドライン）を追加し、ダークモード対応と合わせてキャンバスの視認性を向上させます。

## 目標

- ライト/ダーク/システムの3モード切り替えをUIから行えるようにする
- 選択したテーマをlocalStorageに保存し、リロード後も維持する
- InfiniteCanvasの背景・ドット・基準線がテーマに連動して変化する
- テーマ切り替え時にCSSトランジションでスムーズに遷移する
- アクセシビリティ基準（WCAG AA）のコントラスト比を両モードで満たす
- Storybookでダーク/ライト両モードのプレビューが可能になる
- テーマ関連のユニットテストを整備する

## ユーザーストーリー

### US-001: テーマコンテキストとプロバイダーの作成 [x]

**説明:** 開発者として、アプリ全体のテーマ状態をReact Contextで管理し、どのコンポーネントからでもテーマを参照・変更できるようにしたい。

**受け入れ基準:**

- [x] `src/lib/theme/` にテーマ管理ライブラリを作成
- [x] `ThemeProvider` コンポーネントを作成し、`"light" | "dark" | "system"` の3モードを管理
- [x] `useTheme()` hookを作成し、現在の解決済みテーマ（`"light" | "dark"`）とモード設定（`"light" | "dark" | "system"`）と切り替え関数を返す
- [x] `"system"` 選択時は `matchMedia("(prefers-color-scheme: dark)")` を監視し、OS設定変更にリアルタイムで追従する
- [x] 選択モードをlocalStorageに保存し、初回ロード時に復元する
- [x] SSR/ハイドレーション時のフラッシュ（FOUC）を防ぐため、`<html>` 要素のdata属性（例: `data-theme="dark"`）をスクリプトで早期に設定する
- [x] ユニットテストで以下を検証:
  - デフォルト（localStorage空）ではsystemモードになること
  - localStorage値の保存と復元
  - systemモード時にOS設定変更が反映されること
- [x] 型チェック/lintが通る

## 機能要件

- FR-1: `ThemeProvider` がアプリ全体のテーマ状態を管理し、`"light" | "dark" | "system"` の3モードを提供する
- FR-2: `useTheme()` hookが現在の解決済みテーマ、モード設定、切り替え関数を返す
- FR-3: テーマ選択をlocalStorageに永続化し、リロード後も復元する
- FR-4: `"system"` モード時に `matchMedia` を監視し、OS設定変更にリアルタイム追従する
- FR-5: CSS変数をテーマトークンとして一元管理し、`[data-theme]` セレクタで切り替える
- FR-6: テーマ切り替えUI（3状態トグル: ライト/ダーク/システム）を提供する
- FR-7: テーマ切り替え時にCSSトランジションでスムーズに遷移する
- FR-8: InfiniteCanvasがデフォルト色をCSS変数から取得し、テーマに連動する
- FR-9: InfiniteCanvasにメジャーグリッドライン（基準線）を追加し、テーマに連動する配色とする
- FR-10: InfiniteCanvasにミニマップを追加し、ビューポート位置の把握とクリック/ドラッグによる移動を可能にする
- FR-11: SSR時のFOUC防止スクリプトを `<head>` に挿入する
- FR-12: Storybookにテーマ切り替えデコレーターを追加する
- FR-13: `prefers-reduced-motion` を尊重し、設定時はトランジションを無効化する

## 非目標

- カスタムカラーテーマの作成機能（ユーザーが自由に色を選べる機能）は対象外
- テーマごとのフォント変更は対象外
- ハイコントラストモードは対象外（将来検討）
- サーバーサイドでのテーマ保存（Cookie/DB）は対象外（localStorageのみ）
- 画像やアイコンのダークモード専用差し替えは対象外（filter: invertやopacityで対応）

## デザイン考慮事項

- **ライトモード配色:** 既存の白基調を維持（`#ffffff` 背景、`#171717` テキスト）
- **ダークモード配色:** 純黒は避け、ダークグレー系（`#0a0a0a`〜`#1a1a2e`）を採用。目の疲れを軽減する
- **トグルUI:** ヘッダーまたは画面右上に配置。アイコン（太陽☀/月🌙/モニター🖥）で視覚的に分かりやすく
- **コントラスト:** 両モードでWCAG AAを満たす。特にセカンダリテキスト色に注意
- **キャンバス:** ダークモードのドットと基準線はライトモードより控えめにし、ユーザーがキャンバス上のコンテンツに集中できるようにする

## 技術的考慮事項

- **FOUC防止:** Next.jsの `<head>` にインラインスクリプトを挿入し、`<html data-theme="...">` をレンダリング前に設定する。`next/script` の `beforeInteractive` ストラテジーまたは `layout.tsx` のインラインスクリプトを使用
- **CSS変数アプローチ:** 既存の `prefers-color-scheme` メディアクエリから `[data-theme]` セレクタへ移行する。これにより手動切り替えとOS連動の両立が可能
- **InfiniteCanvasの色取得:** CSS変数をReactから参照するには `getComputedStyle` を使うか、デフォルト値にCSS変数名を使いCSSレベルで解決させる。後者（CSSレベル解決）を推奨
- **Storybook:** `@storybook/addon-themes` または `globalTypes` + デコレーターでテーマ切り替えを実現
- **テスト:** `matchMedia` のモックが必要（Vitestで `vi.stubGlobal` を使用）

## 成功指標

- ダーク/ライト/システムの3モードが正しく切り替わり、リロード後も設定が維持される
- テーマ切り替え時にフラッシュ（FOUC）が発生しない
- 両モードでWCAG AAのコントラスト比基準を満たす
- InfiniteCanvasがテーマ切り替えに連動し、基準線を含む全要素が適切な色になる
- Storybookでダーク/ライト両方のプレビューが可能

- ダークモード時のキャンバス背景色の具体的な値（ユーザーテストで調整が必要か）
- 将来のコンポーネント追加時のテーマトークン拡張方針
