# カット規則とカット除去定理

本ドキュメントでは、カット規則（Cut Rule）の役割とカット除去定理（Cut Elimination Theorem, Hauptsatz）の主張・意義を解説する。

> **前提知識:** シーケント計算（LK）の基本については [01-sequent-calculus.md](./01-sequent-calculus.md) を参照。

## 1. カット規則の形式的定義

### 1.1 規則の形

カット規則（Cut Rule）は以下の形の推論規則である:

```
  Γ ⊢ Δ, A     A, Σ ⊢ Π
──────────────────────────── (Cut)
       Γ, Σ ⊢ Δ, Π
```

ここで:

- `A` を**カット式**（cut formula）と呼ぶ
- 左の前提を**左前提**（left premise）、右の前提を**右前提**（right premise）と呼ぶ
- カット式 `A` は結論のシーケントには出現しない（「切り落とされる」）

### 1.2 カット式の複雑度

カット式の**複雑度**（complexity / degree）は、カット除去の過程で重要な概念である。

**定義:** 論理式 `A` の複雑度 `c(A)` を以下で定義する:

| 論理式                                  | 複雑度                |
| --------------------------------------- | --------------------- |
| 原子式 `P`, `P(t₁, ..., tₙ)`, `t₁ = t₂` | 0                     |
| `¬A`                                    | `c(A) + 1`            |
| `A → B`, `A ∧ B`, `A ∨ B`               | `max(c(A), c(B)) + 1` |
| `∀x.A`, `∃x.A`                          | `c(A) + 1`            |

直感的には、論理式に含まれる論理結合子の「深さ」を数えたものである。

### 1.3 証明のカット階数

**定義:** 証明 `π` の**カット階数**（cut-rank）を以下で定義する:

- `π` にカットが含まれない場合: カット階数は 0
- `π` にカットが含まれる場合: `π` に出現するすべてのカット式の複雑度の最大値

## 2. カット規則の直感的な意味

### 2.1 補題（Lemma）としてのカット

カット規則は**補題を使った証明**に対応する。数学の証明で日常的に行う以下のパターンそのものである:

1. まず、中間的な命題 `A`（補題）を証明する（左前提: `Γ ⊢ Δ, A`）
2. 次に、`A` を既知として使い、最終目標を証明する（右前提: `A, Σ ⊢ Π`）
3. 最終的に、補題 `A` を「切り落として」結論を得る（結論: `Γ, Σ ⊢ Δ, Π`）

**例:** 以下の証明を考える:

```
  ⊢ A     A ⊢ B
────────────────── (Cut)
       ⊢ B
```

これは「まず `A` を証明し、`A` を使って `B` を証明する」ことで `B` を得る。`A` は中間的な補題であり、最終結論 `⊢ B` には出現しない。

### 2.2 Modus Ponens との関係

Hilbert 系の Modus Ponens（MP）:

```
  A     A → B
────────────── (MP)
       B
```

と比較すると、カット規則はより一般的な形の「中間結果の利用」である。MP は特殊な場合として、以下のカットの特殊ケースと見なせる:

```
  ⊢ A     A ⊢ B
────────────────── (Cut)
       ⊢ B
```

ただし、カット規則は前件・後件に任意のコンテキスト（Γ, Δ, Σ, Π）を持てるため、MP よりも一般的である。

### 2.3 カットの必要性と冗長性

カット規則は**便利だが冗長**である。これがカット除去定理の核心的な主張につながる。

**便利さ:** カットがあることで:

- 証明を「モジュール化」できる（補題を分離して証明可能）
- 証明が短く、人間にとって理解しやすくなる
- 段階的な推論が自然に表現できる

**冗長性:** カット除去定理は、カットを使った証明はカットなしの証明に変換可能であることを示す。つまり、カットは証明能力を増やさない。

## 3. カット除去定理（Hauptsatz）

### 3.1 定理の主張

**定理（Gentzen, 1935）:** LK において、カット規則を含む任意の証明は、カット規則を含まない証明に変換可能である。

より形式的に:

> シーケント `S` が LK で証明可能ならば、`S` はカットなしの LK で証明可能である。

すなわち、カット規則は LK の**許容的規則**（admissible rule）である — カットを追加しても新たな定理は増えない。

### 3.2 注意点

- カット除去定理は LK だけでなく LJ（直観主義論理のシーケント計算）でも成り立つ
- カット除去は**構成的**（constructive）に証明される — すなわち、カットを含む証明からカットなしの証明を実際に構築するアルゴリズムが存在する
- カット除去後の証明は元の証明よりも**指数的に大きく**なりうる（後述）

## 4. 定理の意義

カット除去定理は証明論において最も重要な定理の一つであり、多方面に深い帰結をもたらす。

### 4.1 部分式性（Subformula Property）

**部分式性:** カットなしの LK の証明において、証明中に出現するすべての論理式は、結論のシーケントに出現する論理式の**部分式**（subformula）である。

なぜなら:

1. 公理 `A ⊢ A` では `A` は結論の部分式そのもの
2. 構造規則（弱化、縮約）は既存の論理式を操作するだけ
3. 論理規則（¬L, →R, ∧L₁, ...）は結論の主結合子を分解して前提を作るため、前提に現れる論理式はすべて結論の部分式

**唯一の例外がカット規則**である:

```
  Γ ⊢ Δ, A     A, Σ ⊢ Π
──────────────────────────── (Cut)
       Γ, Σ ⊢ Δ, Π
```

カット式 `A` は結論 `Γ, Σ ⊢ Δ, Π` の部分式である必要がない。カット除去により、この例外が取り除かれ、部分式性が保証される。

**部分式性の重要性:**

- 証明に出現しうる論理式の候補が有限になる
- 証明探索を有限の手続きに帰着できる可能性を開く
- 証明の構造に対する強い制約を与える

### 4.2 一貫性の証明（Consistency Proof）

カット除去定理から、LK（および LJ）の**無矛盾性**（consistency）が直接的に導かれる。

**系:** LK は無矛盾である。すなわち、空のシーケント `⊢` は LK で証明不可能である。

**証明の概略:**

1. 仮に `⊢` が LK で証明可能だとする
2. カット除去定理により、`⊢` のカットなし証明が存在する
3. カットなし証明は部分式性を満たす
4. `⊢` には論理式が一つも含まれないので、部分式も存在しない
5. カットなし証明の各規則を調べると、前提なしで `⊢` を導く規則は存在しない
   - 公理は `A ⊢ A` の形であり `⊢` ではない
   - すべての規則は少なくとも1つの論理式を含む
6. 矛盾。よって `⊢` は証明不可能であり、LK は無矛盾 ∎

**意義:** これは Gentzen のオリジナルの動機の一つであった。特に、Peano 算術の無矛盾性証明（Gentzen, 1936）にもカット除去の手法が核心的に使われている。

### 4.3 証明探索への影響

カット除去定理（＝部分式性の保証）は、自動証明探索の理論的基盤を提供する。

**カットありの証明探索の困難:**

結論 `Γ ⊢ Δ` から逆向きに規則を適用して証明を構築する（**ボトムアップ探索**）場合:

- 論理規則の適用は結論から前提を一意的に（または有限の選択肢で）決定できる
- しかし、カット規則を逆向きに適用する場合、カット式 `A` を**推測**しなければならない
- `A` は任意の論理式でありうるため、探索空間が無限になる

**カットなしの証明探索:**

カット除去定理により、カット規則を使わなくても十分であることが保証されるため:

- 探索空間が部分式に限定される
- 結論のシーケントから逆向きに規則を適用すれば十分
- 命題論理の場合、この手続きは**決定可能**である

**応用:**

- **タブロー法**（Tableau Method）: シーケント計算のボトムアップ探索の変形
- **分解法**（Resolution）: カットの特殊ケースに対応する規則に基づく
- **論理プログラミング**（Prolog）: 左側の論理規則のみを使う形式（Horn 節の場合）

### 4.4 補間定理（Craig's Interpolation Theorem）

カット除去定理から、Craig の補間定理を構成的に証明できる。

**補間定理:** `A → B` が恒真（トートロジー）ならば、`A` と `B` の共通の命題変数のみを含む論理式 `C`（**補間式**, interpolant）が存在して、`A → C` と `C → B` がともに恒真である。

カット除去定理の部分式性を利用すると、証明の構造から補間式を構成的に抽出できる。

### 4.5 Herbrand の定理

カット除去定理と部分式性を利用して、Herbrand の定理を証明できる。

**Herbrand の定理（簡略版）:** 存在量化された閉論理式 `∃x₁...∃xₙ.A(x₁,...,xₙ)` が証明可能ならば、有限個の項 `t₁,...,tₖ` が存在して `A(t₁¹,...,tₙ¹) ∨ ... ∨ A(t₁ᵏ,...,tₙᵏ)` が恒真である。

カットなし証明では量化子の例化に使われる項は証明の構造から読み取れるため、Herbrand の定理が自然に導かれる。

## 5. Gentzen のオリジナルの証明

### 5.1 証明の戦略: 二重帰納法

Gentzen のカット除去定理の証明は**二重帰納法**（double induction）に基づく。

**外側の帰納法:** カット式の複雑度 `c(A)` に関する帰納法
**内側の帰納法:** カット規則の「段数」（証明の深さ）に関する帰納法

基本的なアイデアは:

1. 証明中の最も複雑なカット（カット式の複雑度が最大のもの）に注目する
2. そのカットを、より複雑度の低いカット（または段数の短いカット）に変換する
3. この手続きを繰り返すと、最終的にすべてのカットが除去される

### 5.2 変換の主要ケース

カットの両前提の最後の規則（left premise と right premise の最後の推論規則）に応じて場合分けする。ここでは概略のみ示す（詳細は [03-algorithm.md](./03-algorithm.md) で疑似コードとともに解説）。

#### ケース1: 公理とのカット（Axiom Case）

左前提または右前提が公理 `A ⊢ A` の場合、カットは自明に除去できる。

**例（左前提が公理）:**

```
  A ⊢ A     A, Σ ⊢ Π          A, Σ ⊢ Π
──────────────────────  ⟹
     A, Σ ⊢ Π
```

カットの結論 `A, Σ ⊢ Π` は右前提そのものであるから、カットは不要。

**例（右前提が公理）:**

```
  Γ ⊢ Δ, A     A ⊢ A          Γ ⊢ Δ, A
──────────────────────  ⟹
     Γ ⊢ Δ, A
```

カットの結論 `Γ ⊢ Δ, A` は左前提そのものであるから、カットは不要。

#### ケース2: カット式が主式でない場合（Non-principal Case）

カット式 `A` が、直前の規則の**主式**（principal formula）でない場合。すなわち、`A` は直前の規則で「導入」された式ではなく、単にコンテキストとして受け継がれた式である。

この場合、カットを「上に持ち上げる」（pushing up）ことができる。カットの段数が減少するため、帰納法の仮定が適用可能。

**例（左前提で構造規則が最後の場合）:**

```
     Γ ⊢ Δ, A
  ──────────────── (WL)      A, Σ ⊢ Π
   B, Γ ⊢ Δ, A
──────────────────────────────────────── (Cut)
          B, Γ, Σ ⊢ Δ, Π
```

変換後:

```
  Γ ⊢ Δ, A     A, Σ ⊢ Π
──────────────────────────── (Cut)   ← カットの段数が減少
       Γ, Σ ⊢ Δ, Π
    ──────────────── (WL)
     B, Γ, Σ ⊢ Δ, Π
```

カットの段数は元より小さくなるため、帰納法の仮定により除去可能。

#### ケース3: カット式が両方で主式の場合（Principal Case / Key Case）

**最も重要なケース。** カット式 `A` が左前提でも右前提でも主式（導入された式）である場合。

この場合、カット式 `A` の構造に応じた変換を行う。変換後のカット式は元のカット式の**真部分式**であるため、複雑度が減少し、外側の帰納法の仮定が適用可能。

各論理結合子ごとの変換:

**含意のケース: `A = B → C`**

```
     B, Γ ⊢ Δ, C                Γ' ⊢ Δ', B     C, Σ' ⊢ Π'
  ────────────────── (→R)     ──────────────────────────────── (→L)
   Γ ⊢ Δ, B → C               B → C, Γ', Σ' ⊢ Δ', Π'
─────────────────────────────────────────────────────────── (Cut on B → C)
              Γ, Γ', Σ' ⊢ Δ, Δ', Π'
```

変換後:

```
                             Γ' ⊢ Δ', B     B, Γ ⊢ Δ, C
                            ────────────────────────────── (Cut on B)
                                  Γ', Γ ⊢ Δ', Δ, C
                                                           C, Σ' ⊢ Π'
                            ────────────────────────────────────────── (Cut on C)
                                     Γ', Γ, Σ' ⊢ Δ', Δ, Π'
```

ここで、新しいカット式 `B` と `C` はいずれも元のカット式 `B → C` の真部分式であるため、複雑度が厳密に減少する。

**否定のケース: `A = ¬B`**

```
     Γ ⊢ Δ, B                   B, Σ ⊢ Π
  ────────────── (¬R)        ────────────── (¬L)
   Γ ⊢ Δ, ¬B                 ¬B, Σ ⊢ Π
──────────────────────────────────────── (Cut on ¬B)
           Γ, Σ ⊢ Δ, Π
```

変換後:

```
  Γ ⊢ Δ, B     B, Σ ⊢ Π
──────────────────────────── (Cut on B)
       Γ, Σ ⊢ Δ, Π
```

カット式が `¬B` から `B` に変わり、複雑度が 1 減少。

**連言のケース: `A = B ∧ C`**

```
  Γ ⊢ Δ, B     Γ ⊢ Δ, C           B, Σ ⊢ Π   (∧L₁ を使った場合)
────────────────────────── (∧R)    ────────────── (∧L₁)
     Γ ⊢ Δ, B ∧ C                 B ∧ C, Σ ⊢ Π
──────────────────────────────────────────────── (Cut on B ∧ C)
              Γ, Σ ⊢ Δ, Π
```

変換後:

```
  Γ ⊢ Δ, B     B, Σ ⊢ Π
──────────────────────────── (Cut on B)
       Γ, Σ ⊢ Δ, Π
```

カット式が `B ∧ C` から `B` に変わり、複雑度が減少。（∧L₂ の場合は `C` でカットする。）

**選言のケース: `A = B ∨ C`**

```
   Γ ⊢ Δ, B   (∨R₁ の場合)          B, Σ ⊢ Π     C, Σ ⊢ Π
────────────── (∨R₁)              ──────────────────────────── (∨L)
  Γ ⊢ Δ, B ∨ C                     B ∨ C, Σ ⊢ Π
──────────────────────────────────────────────────── (Cut on B ∨ C)
                 Γ, Σ ⊢ Δ, Π
```

変換後:

```
  Γ ⊢ Δ, B     B, Σ ⊢ Π
──────────────────────────── (Cut on B)
       Γ, Σ ⊢ Δ, Π
```

カット式が `B ∨ C` から `B` に変わり、複雑度が減少。（∨R₂ の場合は `C` でカットする。）

**全称量化のケース: `A = ∀x.B`**

```
    Γ ⊢ Δ, B[y/x]                   B[t/x], Σ ⊢ Π
────────────────── (∀R)          ────────────────── (∀L)
  Γ ⊢ Δ, ∀x.B                    ∀x.B, Σ ⊢ Π
──────────────────────────────────────────────── (Cut on ∀x.B)
              Γ, Σ ⊢ Δ, Π
```

変換後: まず `y` に `t` を代入し（∀R の固有変数条件によりこれは安全）、`B[t/x]` でカットする:

```
  Γ ⊢ Δ, B[t/x]     B[t/x], Σ ⊢ Π
──────────────────────────────────── (Cut on B[t/x])
           Γ, Σ ⊢ Δ, Π
```

カット式が `∀x.B` から `B[t/x]` に変わり、複雑度が減少。

**存在量化のケース: `A = ∃x.B`**

全称のケースと対称的。左前提で ∃R、右前提で ∃L を使用する場合:

```
    Γ ⊢ Δ, B[t/x]                   B[y/x], Σ ⊢ Π
────────────────── (∃R)          ────────────────── (∃L)
  Γ ⊢ Δ, ∃x.B                    ∃x.B, Σ ⊢ Π
──────────────────────────────────────────────── (Cut on ∃x.B)
              Γ, Σ ⊢ Δ, Π
```

変換後: `y` に `t` を代入し、`B[t/x]` でカットする:

```
  Γ ⊢ Δ, B[t/x]     B[t/x], Σ ⊢ Π
──────────────────────────────────── (Cut on B[t/x])
           Γ, Σ ⊢ Δ, Π
```

#### ケース4: 構造規則との相互作用

カットの直前の規則が構造規則（弱化、縮約）の場合、カットを構造規則の上に「持ち上げる」ことで段数を減少させる。

**弱化とカット:**

カット式が弱化で導入された場合（カット式は実際には「使われていない」）:

```
       Γ ⊢ Δ
  ────────────── (WR)      A, Σ ⊢ Π
   Γ ⊢ Δ, A
──────────────────────────────────── (Cut)
        Γ, Σ ⊢ Δ, Π
```

この場合、`A` は弱化で追加されただけなので、カットは実質的に不要。変換:

```
  Γ ⊢ Δ            Σ ⊢ Π
  (弱化で Σ, Π を追加)
  Γ, Σ ⊢ Δ, Π
```

（複数回の弱化適用で結論に到達。）

**縮約とカット:**

```
  Γ ⊢ Δ, A, A                A, Σ ⊢ Π
────────────── (CR)
   Γ ⊢ Δ, A
──────────────────────────────────────── (Cut)
        Γ, Σ ⊢ Δ, Π
```

変換後:

```
  Γ ⊢ Δ, A, A     A, Σ ⊢ Π
──────────────────────────── (Cut on first A)    ← 段数減少
       Γ, Σ ⊢ Δ, A, Π
                            A, Σ ⊢ Π
──────────────────────────────────── (Cut on remaining A)    ← 段数減少
       Γ, Σ, Σ ⊢ Δ, Π, Π
       (縮約で整理)
       Γ, Σ ⊢ Δ, Π
```

### 5.3 停止性の議論

**なぜアルゴリズムは停止するか?**

二重帰納法の構造により:

1. **外側の帰納法（カット式の複雑度）:** ケース3（principal case）の変換では、新しいカット式の複雑度が元のカット式の複雑度より**厳密に小さい**。これは新しいカット式が元のカット式の真部分式であることによる。

2. **内側の帰納法（カットの段数 / 証明の深さ）:** ケース2（non-principal case）とケース4（構造規則）の変換では、カット式の複雑度は変わらないが、カットが証明の「上方」に移動するため、カットの上にある証明の深さが減少する。

3. **基底ケース:** ケース1（公理とのカット）に到達すると、カットは自明に除去される。

これらの組み合わせにより、変換手続きは必ず停止する。具体的には:

- ケース3が適用されるたびに、外側の指標（複雑度）が減少
- ケース2・4が適用されるたびに、内側の指標（段数）が減少
- いずれの指標も自然数であるから、減少列は有限で停止

したがって、有限回の変換でカットなし証明に到達する。

### 5.4 証明のサイズ増大

カット除去によって証明のサイズは**劇的に増大**しうる。

**命題論理の場合:** カットを含む長さ $n$ の証明からカットを除去すると、結果の証明の長さは $2^{2^{...^{n}}}$（指数の塔、$n$ 段）程度になりうる。すなわち、非初等的（super-exponential）な爆発が起こる。

**一階述語論理の場合:** さらに悪化しうる。Statman (1979) はカット除去による証明の伸長が非初等的であることを示した。

**直感的な理由:** ケース3（principal case）の変換では、カットが2つの新しいカットに分裂する場合がある（例: 含意のケースで `B → C` のカットが `B` のカットと `C` のカットに分裂）。さらに、ケース4の縮約との相互作用で証明が複製されるため、サイズが急激に増大する。

この証明サイズの爆発は、カット除去の**本質的な**性質であり、アルゴリズムの工夫では回避できない（下界の存在が証明されている）。

## 6. Mix 規則とMix 除去

カット除去の証明をよりクリーンに行うため、**Mix 規則**（mix / multicut）を導入する定式化がある。

### 6.1 Mix 規則の定義

```
  Γ ⊢ Δ, A, ..., A     A, ..., A, Σ ⊢ Π
──────────────────────────────────────────── (Mix)
              Γ, Σ ⊢ Δ, Π
```

Mix 規則はカット規則の一般化で、カット式 `A` が左前提の後件と右前提の前件に**複数回**出現する場合に、すべての出現を一度に切り落とす。

### 6.2 Mix 除去の利点

Mix 規則を使うと、縮約規則との相互作用が簡素化される:

- カット除去ではケース4の縮約処理が複雑になりがち
- Mix 除去では、縮約を「Mix の中に吸収」できるため、場合分けがシンプルになる

Troelstra & Schwichtenberg (2000) は Mix 除去に基づく証明を推奨している。

## 7. カット除去の限界と発展

### 7.1 カット除去が成り立たない体系

すべてのシーケント計算でカット除去が成り立つわけではない:

- 公理スキーマとしてシーケント `Γ ⊢ Δ` を追加した体系では、カット除去が失敗しうる
- **解決策:** 公理スキーマを適切な規則に変換する（例: 初期シーケントの拡張 `Γ, A ⊢ A, Δ`）

### 7.2 一階算術とカット除去

Peano 算術 (PA) のシーケント計算定式化では、数学的帰納法の公理スキーマがあるため、単純なカット除去は適用できない。Gentzen (1936, 1938) は超限帰納法（ε₀ までの超限帰納法）を用いて PA の無矛盾性を証明した。

### 7.3 現代の発展

- **線形論理のカット除去:** Girard (1987) の線形論理でもカット除去が成り立つ。構造規則の制限により、証明の動的な挙動をよりきめ細かく分析できる
- **カリー＝ハワード対応:** カット除去は計算（β簡約）に対応する。カットなし証明は正規形に対応する
- **証明ネット:** 線形論理の証明ネットにおけるカット除去は、グラフ書き換えとして定式化される
